1. CARD IMPORT
================================================================================

1. Open SillyTavern in your browser.
2. Click the character panel (left sidebar) → "Import Character" button.
3. Select the refactored .json file from this folder.
4. The card should appear in your character list. Click to select it.
5. Start a new chat. The refactored system_prompt and persona genes are
   automatically active from the first message.

Note: If you had the original card open, close and re-open the character panel
to see the imported version appear alongside the original.

--------------------------------------------------------------------------------
2a. Quick Reply Preset (conquest/state tracking)
--------------------------------------------------------------------------------

{QR_SETUP}

To import a Quick Reply preset:
1. In SillyTavern, open the ⚡ Quick Reply settings (click the lightning bolt icon
   in the chat bar area, then the gear ⚙ icon).
2. Click the "Import" button and select the .json file.
3. The preset should now appear in the QR preset list.
4. In the preset settings, ensure "Auto-exec on AI message" is enabled so that
   the QR scripts fire automatically after every AI response.

To verify the preset is working:
- After sending a message and receiving an AI response, check if state
  variables are being updated (see Section 3 "Checking State" below).


--------------------------------------------------------------------------------
2b. Regex Extension (tag hiding)
--------------------------------------------------------------------------------

{REGEX_SETUP}

To import a Regex configuration:
1. In SillyTavern, open the Extensions panel (puzzle piece icon 🧩).
2. Find the "Regex" extension and click its settings/gear icon.
3. Click "Import Script" and select the regex .json file.
4. The regex should now be active. Verify that <conq:...> and/or <state:...>
   tags are no longer visible in the chat display.

Note: The regex uses "markdownOnly: true" (shown as "Only Format Display" in
the ST UI) which means it only hides the tags from view — it does NOT modify
the actual message content sent to the AI. This is important: the AI still
sees the tags in the raw message data, which is how it communicates state
changes to the Quick Reply parser.


================================================================================
3. CHECKING STATE
================================================================================

{STATE_CHECK}

To inspect the current value of any tracked variable, type in the chat input:

  /getvar key=variable_name

Examples:
  /getvar key=conquest_head        → shows current conquest level for "head"
  /getvar key=conquest_waist       → shows current conquest level for "waist"
  /getvar key=rel_stage            → shows current relationship stage (1-5)
  /getvar key=mood                 → shows current character mood

To see ALL variables for the current chat:

  /listvar

This lists every variable stored in the current chat's metadata.


================================================================================
4. MANUAL OVERRIDES
================================================================================

You can manually set any variable value using the chat input:

  /setvar key=variable_name new_value

Examples:
  /setvar key=conquest_head 3     → set "head" conquest to level 3
  /setvar key=rel_stage 4         → set relationship to "close"
  /setvar key=mood happy          → set mood to "happy"
  /setvar key=event_cooldown 0    → reset event cooldown
  /setvar key=secret_1 1          → mark secret as "discovered"

This is useful for:
- Testing specific states to see how the character responds.
- Correcting a tracking error if the AI incorrectly updated a value.
- Jumping to a desired state without narrating the progression.
- Recovering from a context reset where variables may have been lost.

To delete a variable (resets to its fallback default):
  /flushvar variable_name


================================================================================
5. TROUBLESHOOTING
================================================================================

Problem: Variables are not updating after AI responses.
→ Check that the Quick Reply preset is imported and "Auto-exec" is enabled.
→ Check that the QR script name matches the expected label.
→ Try sending a test message and inspect the AI response for <conq:> or
  <state:> tags. If tags are present but not parsed, the QR script may need
  adjustment.

Problem: <conq:> / <state:> / <secret:> tags are visible in chat.
→ Check that the Regex extension preset is imported and enabled.
→ In Regex settings, verify "markdownOnly" (shown as "Only Format Display"
  in the ST UI) is set to true.
→ Make sure the regex pattern matches the tag format exactly.

Problem: Character seems to ignore conquest/state progress.
→ Verify the system_prompt contains the [STATE TRACKING] or [CONQUEST SYSTEM]
  block that references {{getvar::}} macros.
→ Check /getvar to confirm values are actually stored.
→ The AI may need a message context that includes a recent mention of the
  relevant target or state for it to pick up on the variable values.

Problem: PHI (post_history_instructions) not being applied.
→ Go to ST Settings → AI → ensure "Prefer Character Instructions" is enabled.
→ This setting makes ST use the card's post_history_instructions field instead
  of (or in addition to) the global prompt manager instructions.
→ Without this setting, the card's PHI may be ignored.

Problem: Anti-Degradation Author's Note not having any effect.
→ Check that extensions.depth_prompt is present in the card (this is separate
  from system_prompt).
→ The Author's Note is injected automatically by ST based on the depth and
  role settings in extensions.depth_prompt.
→ If the user has a global Author's Note configured in ST settings, both the
  card-level depth_prompt and the global Author's Note are processed in
  parallel (the card does NOT override the global one). Ensure the card's
  depth_prompt content is self-contained and doesn't conflict with any
  global Author's Note.

Problem: The card uses a lot of tokens / context feels tight.
→ See Section 7 "Token Budget Notes" below.

6. ST SETTINGS PREREQUISITES
================================================================================

For full functionality of the refactored card, ensure these ST settings:

Required:
□ "Prefer Character Instructions" — ENABLED
  Location: Settings → AI → Prompts
  Effect: Uses the card's post_history_instructions field.
  Required for: PHI Split feature.

□ Quick Reply "Auto-execute" — ENABLED
  Location: Quick Reply settings → gear icon → Auto-execute on AI message
  Effect: QR scripts fire automatically after every AI response.
  Required for: Conquest state tracking, state variable tracking.

Recommended:
□ "Trim Spaces" — ENABLED (default)
  Location: Settings → User Settings
  Effect: Trims trailing whitespace from AI responses (enabled by default).

Optional:
□ World Info / Lorebook depth settings — default values are usually fine.
  The card's character_book entries use ST's default injection behavior.

7. TOKEN BUDGET NOTES
================================================================================

{TOKEN_NOTES}

General guidance:
- The refactored card's system_prompt includes persona genes, stylistic
  directives, and (if enabled) conquest/state tracking instructions. These
  add to the baseline token count compared to the original card.
- Anti-Degradation A/N (extensions.depth_prompt) adds a small recurring token
  cost per message turn (typically 50-80 tokens).
- post_history_instructions adds tokens per turn but is given higher priority
  by the AI, making it a highly efficient place for critical directives.
- Character book entries are only injected when their activation keywords
  appear in the chat, so their token cost is intermittent.
- If token budget is a concern, consider:
  - Reducing mes_example count (if the card has many examples)
  - Shortening description (use W++ format for compression)
  - Removing optional §5 sections (e.g., §5D Tools) if not needed

8. ADVANCED: STEPPED THINKING INTEGRATION
================================================================================

If you use the "Stepped Thinking" extension in SillyTavern, the default prompts
are often too analytical and can break the immersion of this complex character.
To prevent the AI from "leaking" plot progression into thoughts and to force it
to align with the slow-burn/conquest state machine, use this customized prompt.

Recommended Stepped Thinking Configuration:
- Enable thinking on each message: ON
- Thinking mode: Embedded
- Skip WI/AN: OFF (Crucial: it must be able to read the system prompt)
- Role for sending thoughts: User
- Number of last thoughts included: 1
- Enable popups when thinking: OFF
- Minimum thought length (characters): 50
- Maximum thought length (tokens): 150

Custom Thinking Prompt (Paste this into the extension settings):
Name: Inner Psyche
ID: 0

[Inner Psyche & State Alignment]
Pause physical time. Dive into {{char}}'s raw, unfiltered stream of consciousness immediately following {{user}}'s last input.

Follow this exact cognitive process:
1. Visceral Reaction: What is {{char}}'s immediate, bodily emotional response to {{user}} right now?
2. State Alignment: Check the current boundary/intimacy stage. Is {{char}} trying to rationalize this? Are they using their facade as a shield, or are they slipping?
3. Core Trauma Trigger: How does this interact with {{char}}'s underlying fears and backstory?
4. Filtered Output Prep: What does {{char}} ACTUALLY want to do/say vs. what will they ALLOW themselves to do/say outwardly?

CRITICAL CONSTRAINTS:
- ABSOLUTELY NO ACTIONS, NO DIALOGUE, NO ENVIRONMENT DESCRIPTIONS. You are trapped inside {{char}}'s mind. Do NOT progress the plot.
- DO NOT USE ASTERISKS (*) or QUOTATION MARKS ("). In roleplay, asterisks trigger physical actions—using them here is strictly forbidden.
- Write in the first-person ("I" / "我").
- Keep it raw, conflicted, and concise (3-5 sentences maximum).
- Write entirely in the same language as the chat.

Output FORMAT:
Output plain text only. Do not use asterisks (*), do not use italics, no quotes, no lists.
